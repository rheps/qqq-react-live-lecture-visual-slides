# Structure Catalog — 내용 모양 → 구조 끄집어내기

이 문서는 "내용을 보고 **어떤 모양의 구조**에 담을지"를 고르는 카탈로그다.
`component-selection-rules.md`의 큰 intent 표는 **이상적 목표**(아직 안 만든 것 포함)를 나열한 것이고,
이 파일은 **실제로 만들어져 검증된 구조**와 **내용 모양**을 1:1로 잇는다.

> 핵심 실수: 내용이 흐름·위계·대비인데 **전부 카드로** 만든다.
> 카드는 "위계 없는 병렬 묶음"에만 쓴다. 내용의 모양이 먼저고, 구조는 그 모양을 따라간다.

---

## STEP 0 — 내용의 모양부터 그린다 (구조 선택 전 필수)

슬라이드 데이터를 쓰기 전에, 그 덩어리가 **어떤 모양인지** 한 문장으로 말한다.
모양이 정해지면 구조는 거의 자동으로 결정된다.

| 내용이 이런 모양이면… | 쓰는 구조 (✅=구현됨) | 절대 이렇게 하지 마라 |
|---|---|---|
| **A가 B를 거쳐 C로 이어지는 메커니즘/변환** | `flow` ✅ | 카드 3장으로 나열 |
| **차례·단계 (1→2→3, 시간 순서)** | `vertical-step-flow` ✅ | 카드, 불릿 |
| **둘의 대비 (소비 vs 생산, 전 vs 후)** | `icon-versus` ✅ | 카드 2장 |
| **한 문장 주장 (펀치라인)** | `key-message` (+`hl` 핵심어) ✅ | 제목+본문 단락 |
| **던지는 질문 하나** | `big-question` ✅ | key-message |
| **위계 없는 병렬 묶음 3~4개** | `four-card` / 3-keyword ✅ | 흐름·화살표 억지로 넣기 |
| **동등한 세 요소가 모여 하나 (삼위일체/통합)** | `trinity` ✅ | 카드 3장 평면 나열 |
| **수치 비교·순위** | `animated-bar-chart` ✅ | 글자로 "30%, 50%…" 나열 |
| **항목 × 속성 표** | `table-slide` ✅ | 카드 여러 장으로 쪼개기 |
| **흩어진 용어 무리 (관계 약함)** | `keyword-cluster` ✅ | 표, 카드 |
| **영상** | `video` ✅ | 캡처 이미지 |
| **부/장 시작, 표지** | `section-divider` / `hero-title` ✅ | key-message |
| **마무리·행동 촉구** | `closing-prompt` ✅ | key-message |
| **상위→하위로 가지치는 위계/분기** | `tree` 🔨 (없으면 만든다) | 카드로 평평하게 펴기 |
| **한 중심에서 여러 갈래로 방사** | `hub-spoke`/mindmap 🔨 (3노드면 `trinity` ✅) | 카드 나열 |
| **원인 → 결과 사슬** | `cause-effect-chain` 🔨 (flow 변형) | 카드 |
| **연표 (시간 위의 점들)** | `timeline-dots` 🔨 | 표 |
| **2축 분류 (4분면)** | `2x2-matrix` 🔨 | 카드 4장 |

🔨 = 아직 템플릿이 없다. 내용이 그 모양을 **진짜로 요구하면** 카드로 우회하지 말고 **새로 만든다** (slide 15에서 `flow`를 새로 만든 것처럼). 만들기 전 사용자에게 "이 모양엔 X 구조가 맞고 지금 없으니 만들겠다"고 알린다.

## A. 구현된 구조 14개 (✅ 실제로 끄집어 쓸 수 있는 것)

1~13은 reference 덱 `kangui-1-consumer-to-producer.html`, 14(`trinity`)는
`촉법소년-논증게임-온라인진행-슬라이드.html`에 실제로 들어있고 검증된 구조다.
`case "..."` 디스패처에 등록된 그대로다.

### 1. `flow` — 가로 흐름 / 메커니즘
내용 모양: **A → (변환) → B**. 입력이 어떤 장치를 거쳐 결과로 바뀌는 과정.
```
 🤖            ⚖️ [box]          👥
 A  ──→   ┌──────────┐   ──→    C
 입력      │ 변환 장치 │          결과
          │ ⓐ …  ⓑ … │
          └──────────┘
            (caption: 출처·주장)
```
- `stages:[{emoji,label,sub}, {emoji,label,items:[...]}, {emoji,label,sub}]`
- **가운데(items 가진) 노드만 glass-card로 박아 강조** — 거기가 메커니즘의 심장.
- 양끝 노드는 emoji+label만(plain). 사이에 `→`(active 시 등장).
- `caption`: 하단 출처/주장. `builds:[1,2,3]` 순차 등장.
- 워크드 예: slide 15 = "AI가 부 생산 → 분배장치(AI세/UBC) → 모두에게 기본소득, 주장: 알트먼·머스크".

### 2. `vertical-step-flow` — 세로 단계
내용 모양: **차례/순서/방법론**. 1단계 다음 2단계.
- `steps:[{emoji,label,sub}]`, `wide = steps.length<=3` → 간격 넓힘(`stepGap`).
- 원과 라벨, 사이 세로 커넥터. `titleSmall`로 제목 줄이기 옵션.
- 5부 강의 구조, 3단계 생산자 교사 등.

### 3. `icon-versus` — 둘의 대비
내용 모양: **A vs B**, 또는 **A에서 B로의 전환**.
- `left`/`right` 각각 `{emoji,title,items}`; `side`(정사각 px)로 카드 크기, `fontScale`.
- 한쪽을 작게/다른 쪽을 크게 해서 **무게 차이**를 줄 수 있다(기존 연수↔오늘 연수).
- `caption`: 마지막 build에 하단 한 줄(대비의 결론, `hl` 적용). "소비에서 생산으로의 전환".

### 4. `key-message` — 한 문장 주장
내용 모양: **펀치라인 한 줄**.
- `message`(크게, `hl`로 핵심어 그라디언트) + 선택 `lead`(위 작은 muted 줄, `hl` 적용).
- 셋업은 `lead`(작게), 핵심은 `message`(크게) — **강조 역전 방지의 핵심 도구**.

### 5. `big-question` — 질문 하나
내용 모양: **청중에게 던지는 질문**. 큰 물음표/문장 중심.

### 6. `four-card` — 병렬 묶음 (위계 없음)
내용 모양: **동등한 항목 3~4개**. 순서·인과·위계가 **없을 때만**.
- 흐름이나 대비를 카드로 만들지 말 것. 그건 위 flow/versus로.

### 7. `animated-bar-chart` — 수치 비교
내용 모양: **양·비율·순위**. 막대가 자라며 등장.

### 8. `table-slide` — 표
내용 모양: **항목 × 속성** 격자. 발표형은 4행4열 이하로.

### 9. `keyword-cluster` — 용어 무리
내용 모양: **관계가 느슨한 용어 여러 개**가 한 주제 아래 흩뿌려짐.

### 10. `video` — 영상
- YouTube 임베드(`videoId`). `vertical`(쇼츠 9:16, `vh`로 높이), `crop`(가운데 띠만), `start`/`end`(구간 재생).
- `file://`에선 오류 153 — localhost/배포에서만 재생.
- 글 없이 영상만: `title` 비우고 영상만.

### 11. `section-divider` / 12. `hero-title` — 표지·구분
- `hero-title`: 덱 표지. 제목에 `hl`로 핵심어 그라디언트("소비자에서 *생산자*로").
- `section-divider`: 부/장 전환.

### 13. `closing-prompt` — 마무리
내용 모양: **행동 촉구·다음 질문**으로 닫기.

### 14. `trinity` — 삼위일체 삼각형 (셋이 하나로)
내용 모양: **동등한 세 요소가 모여 하나의 상위 개념**을 이룬다. 단순 병렬이 아니라 "셋 → 하나"의 통합이 핵심일 때.
```
            🔍 사실성              (위 꼭짓점)
           /    \
          / 스포크 \
   🧲 ───[🛡️ 탄탄한 반론]─── ⚖️   (중앙 허브 + 왼·오른 꼭짓점)
  관련성    (셋이 모임)    충분성
        └──── 둘레선 ────┘
```
- `items:[{emoji,label,desc}]` 정확히 3개 + `core`/`coreEmoji`(중앙 허브, 옵션).
- 꼭짓점 3장이 삼각형(둘레선)을 이루고, **마지막에 허브가 등장하며 색 스포크가 빛을 내며 모여든다**("삼위일체 방패" 구조).
- `four-card`(평면 병렬)와의 차이: **통합 메시지(core)**가 있고 셋의 *합쳐짐*을 강조한다. `hub-spoke`의 3노드 특화판.
- 워크드 예: slide c15 = "사실성·관련성·충분성 → 탄탄한 반론".
- **전체 코드·좌표·스키마는 `trinity-template.md`** 참고.

공통 장식: `Eyebrow`(페이드 라인 + 자간 넓힌 작은 라벨) — 칩/알약 대신 슬라이드 상단 분류표시. light/dark/warm 테마.

## B. 아직 없는 구조 (🔨 내용이 요구하면 만든다)

reference 덱엔 아직 없다. 내용 모양이 이걸 **진짜로 요구하면** 카드로 뭉개지 말고 새로 만든다.
만들 때는 `flow`를 만든 방식과 같다: 모듈 레벨 컴포넌트 + 디스패처 `case` 등록 + `builds` 순차 등장 + spring-pop.

| 구조 | 내용 모양 | 스케치 | 만들 때 핵심 |
|---|---|---|---|
| `tree` | 상위 개념이 하위로 **가지치기**(분류 위계) | `루트→[자식,자식→[손자]]` | 세로 또는 가로 트리, 부모-자식 커넥터 라인 |
| `hub-spoke`/`mindmap` | **한 중심**에서 여러 갈래 방사 | `중심●←→가지×N` | 중심 노드 + 방사 배치, 가지마다 라벨 |
| `cause-effect-chain` | **원인→결과** 연쇄 | `원인 ⇒ 결과 ⇒ 결과` | `flow`의 변형(화살표 강조, 노드 박스 없음) |
| `timeline-dots` | **연표**(시간 위 사건들) | `──●──●──●──` | 가로 축 + 점 + 연도/사건 라벨 |
| `2x2-matrix` | **2축 4분면** 분류·우선순위 | `사분면 격자` | 축 라벨 2개 + 4칸, 항목 배치 |
| `loop/cycle` | **반복 개선 루프** | `①→②→③→①` | 원형 배치 + 되돌아오는 화살표 |

> 판단 기준: "이 내용은 위계인가? 방사인가? 인과 사슬인가?"가 **명확히 yes**면 카드로 평평하게 펴지 말고 그 구조를 만든다. 애매하면(그냥 병렬 묶음이면) `four-card`가 맞다.

## C. 워크드 예시 — slide 15 (어떻게 구조를 끄집어냈나)

원본 내용:
> 보편적 기본소득(UBI). AI가 부를 생산 → 분배 장치(ⓐ AI세, ⓑ Universal Basic Compute) → 모두에게 기본소득. 주장: 샘 알트먼·일론 머스크.

1. **모양 판정**: "A(생산)가 → 장치(분배)를 거쳐 → C(기본소득)"로 흐르는 **메커니즘**. 위계도, 단순 병렬도 아님.
2. **구조 선택**: 메커니즘 → `flow`. (카드 4장이 첫 시안이었는데 사용자가 "카드 말고 다른 구조" 요청 → 모양에 맞는 flow로 교체.)
3. **강조 배치**: 메커니즘의 심장인 **분배 장치**(AI세/UBC)를 가운데 노드로 **박스 강조**. 양끝(생산·기본소득)은 plain. 주장(인명)은 하단 caption으로.
4. 결과: 내용의 모양과 화면의 모양이 일치. 청중이 "무엇이 무엇을 거쳐 무엇이 되는지"를 한눈에.

이 과정이 이 카탈로그가 하려는 전부다: **내용의 모양을 말로 먼저 그리고 → 그 모양의 구조를 끄집어낸다.**
