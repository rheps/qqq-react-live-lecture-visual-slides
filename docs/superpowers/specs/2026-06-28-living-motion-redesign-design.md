# 살아있는 모션 재설계 — 설계 문서

작성일: 2026-06-28

## 1. 문제

`component-gallery.html`의 모션 엔진은 모든 등장 키프레임(`aRiseT`, `aRiseB`, `aSlideL/R`,
`aPop`, `aFade`, `aGrowY/X`, `aDraw`, `aBloom`, `aWipeX`, `aGrowC`, `aPush`, `aSweepSettle`,
`aQMark`)을 `rep="infinite"`로 돌리고, 각 키프레임이 `100%`에서 **숨김 상태**(`opacity:0` /
`scale(0)` / `dashoffset=len`)로 돌아간다. 그 결과 **75개 전 컴포넌트**가 예외 없이
`등장 → 정착(13~86%) → 사라짐 → 재등장`을 무한 반복한다.

이것이 사용자가 인식한 "만들어지고, 사라지고, 다시 만들어지는" 문제이며 **전역적**이다.
일부 컴포넌트는 깨짐(레이아웃 재계산·draw 재실행)까지 발생한다.

목표: **컴포넌트는 등장 1회 후 정착해 영원히 사라지지 않되, 그 위에 비파괴적인 "생명" 모션이
계속 돈다.** ("39번처럼 — 사라지지 않지만 계속 살아 움직이는 느낌")

## 2. 해결 구조 — 2층

### 1층: 전역 메커니즘 수정 (75개 전부)

- 모든 등장 모션을 **1회성**으로: `rep=1`(또는 `repeatCount=1`), `fill="forwards"`.
- 등장 키프레임에서 **`100% → 숨김` 꼬리를 제거**하고, 키프레임이 **정착 상태에서 끝나게** 한다.
  - 예: `aRiseT`를 `0%{opacity:0;translateY(-15px)} 100%{opacity:1;none}`로 (정착에서 종료).
  - 기존 키프레임을 직접 고치지 말고 **`*-in` 1회용 키프레임 세트**(`aRiseTIn` 등)를 새로 만들어
    등장에 쓰고, 무한 루프 키프레임은 "생명 레이어" 전용으로만 남긴다.
- 정착 후에는 컴포넌트가 **영구히 그대로 유지**된다(절대 opacity→0 없음).

### 2층: 생명 레이어 (비파괴적 영구 루프)

정착된 도형 **위/안/옆**에서만 도는 무한 모션. 도형·구조·글씨를 절대 사라지게 하지 않는다.
읽는 글씨에는 지속 모션 금지(SKILL 9번) — 글씨 주변 장식이나 비텍스트 도형에만 건다.

`reduced-motion`에서는 1층 등장은 즉시 정착, 2층 생명 레이어는 전부 정지.

## 3. 효과 메뉴 (생명 레이어 어휘 14 + 정지)

1. 강조 순회 — 조각·카드·층을 하나씩 차례로 강조(색/글로우/스케일), 무한 반복
2. 양쪽 번갈아 강조 — 두(또는 네) 영역이 번갈아 빛남
3. 선 위 흐르는 점 — 선/경로 위를 점·빛이 영구히 흐름(SMIL animateMotion)
4. 축 따라 이동 마커 — 글로우 마커가 축을 따라 한 방향으로 영구 이동
5. 방향 펄스 — 화살표/갈매기/마디가 순서대로 톡톡 빛남
6. 글로우 호흡 — 뒤 헤일로/테두리가 은은히 커졌다 작아짐(글씨 정지)
7. 맥박 — 핵심 요소가 심장박동처럼 살짝 커졌다 작아짐
8. 파동 펄스 — 중심에서 링이 바깥으로 퍼지는 파동 반복
9. 반짝임 — 작은 스파클이 깜빡이며 떴다 사라짐(점·정답 안 등)
10. 광택 스윕 — 빛줄기가 도형/막대 위를 스윽 지나감
11. 미세 부유 — 위아래로 살짝 둥실
12. 흔들림 — 좌우로 갸우뚱
13. 회전 — 계속 돌아감
14. 시소/진자 — 균형추가 흔들
0. 효과 없음 — 등장 후 완전 정지

> 시안: `_effect-menu-demo.html` (14개 효과 라이브 데모)

## 5. 구현 범위와 단일 진실원천

- **단일 진실원천: `component-gallery.html`** (모양·모션 원본). 여기서 키프레임/`SHOW`/
  `generic`/`chart`를 고친다.
- 새 생명 효과는 재사용 가능한 헬퍼로 정리: `cycleHighlight`, `altEmphasis`, `flowDot`(기존),
  `axisMarker`, `dirPulse`, `breatheGlow`, `pulse`, `ripple`, `sparkle`, `sheen`, `floatBob`,
  `wobble`, `spin`(기존), `seesaw`(기존 39 로직). 각 컴포넌트는 결정 표대로 1~2개 호출.
- 수정 후 **`node references/build-kit.js`**로 `references/qqq-component-kit.js` 재생성
  (키트는 파생물 — 직접 수정 금지).
- 문서 갱신: `references/component-motion.md`(번호별 시그니처 표), 필요 시 `SKILL.md` /
  `references/motion-system.md`에 "등장 1회 + 생명 레이어" 원칙 명시.
- 가드/테스트: `tests/gloss-guard.mjs` 통과 유지. 가능하면 "등장 키프레임에 100%→opacity:0
  꼬리가 없다"를 검증하는 가드 추가.

## 6. 검증

- 갤러리를 로컬 서버로 띄워 **모든 컴포넌트가 정착 후 사라지지 않는지** 육안 확인
  (특히 ⚠️ 5종과 차트 전반).
- `prefers-reduced-motion`에서 전부 정지·정착하는지 확인.
- 실제 슬라이드 캔버스(1200×900)에서 글씨 24px+ 유지, 읽는 글씨에 지속 모션 없는지 확인.

## 7. 비목표 (YAGNI)

- 새 컴포넌트 추가·디자인 변경 없음. 모션 생명주기와 생명 레이어만 다룬다.
- React/Framer Motion 등장 래퍼(`MotionItem`)는 그대로 — 키트 내부 노드 모션만 손댄다.
